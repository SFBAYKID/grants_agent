# Grant Product Ideas

These are **assumed** product hypotheses, not implemented capabilities. Each needs user
validation, architecture review, and tests before being described as available.

1. **Reason-to-call brief:** summarize the verified funding event, likely security
   scope, active window, official source, CRM context, and two tailored discovery
   questions in one compact card.
2. **Buying-committee map:** find and verify Technology, Facilities, Operations,
   Business, Superintendent/City Manager, and procurement contacts; show gaps instead
   of inventing people.
3. **Board-agenda early warning:** detect agenda items approving grant applications,
   security designs, bid releases, and vendor awards before a formal RFP appears.
4. **Funding-to-RFP timeline:** connect application opening, board approval, award,
   spending window, RFP, addenda, award notice, and contract start as distinct events.
5. **Territory watchlists:** let each rep follow states, counties, named accounts,
   programs, product categories, or competitors without changing global notifications.
6. **Opportunity change alerts:** notify only meaningful changes—deadline extensions,
   addenda, cancellation, award, amount changes, or a new named recipient.
7. **Duplicate-agency resolver:** identify aliases and parent/child entities while
   preserving source names and requiring human review for ambiguous merges.
8. **Relationship-aware ranking:** boost verified active Salesforce Accounts and open
   Opportunities, but never boost stale, ambiguous, or unavailable CRM results.
9. **Engagement learning:** adjust program/source ranking from opens, thread replies,
   contact requests, exports, CRM additions, meetings, and disqualifications—with
   bounded weights and an audit trail rather than opaque self-modification.
10. **Negative-feedback taxonomy:** capture stale, wrong entity, irrelevant scope,
    duplicate, existing vendor, no budget, and bad timing so each failure improves a
    specific stage.
11. **Coverage confidence:** show whether a result set covers a complete official feed,
    a partial portal, indexed records, or a live web fallback.
12. **RFP requirement extractor:** pull mandatory walkthroughs, deadlines, insurance,
    certifications, submission portal, question deadline, and requested equipment from
    official documents with page/section citations.
13. **Incumbent and award history:** connect prior public contract awards and board
    approvals to likely replacement cycles without asserting an incumbent when the
    evidence is incomplete.
14. **Cooperative-contract fit:** identify whether a buyer can use an applicable public
    cooperative contract, while clearly separating eligibility from a verified buying
    decision.
15. **Human handoff memory:** preserve which result set, contact, source, Salesforce
    record, and outreach draft a thread refers to so “that one,” “the other two,” and
    “draft it again” remain deterministic.
16. **Quiet-mode intelligence:** keep monitoring while suppressing ordinary alerts;
    interrupt quiet mode only for a user-defined exceptional threshold.
17. **Morning opportunity queue:** provide a private, on-demand ranked work queue—not a
    channel digest—with reasons each lead moved up or down.
18. **Source-health dashboard:** expose last successful poll, freshness, row count drift,
    parser warnings, portal access boundary, and live-verification state per source.
